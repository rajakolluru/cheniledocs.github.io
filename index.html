Basics

Chenile is a Microservices development framework in Java. It generates code and contains important libraries that aid in software development.

Chenile was conceived as an INVM interception framework that allows programmers to write code without worrying about other horizontal concerns. This code would then automatically get exposed via multiple transports (like HTTP, KAFKA etc.).

Over time it has evolved to become much more than that. Chenile has a collection of patterns, blue prints, testing strategies and some best practices.

Chenile standardizes the frameworks that needs to be used. It also standardizes the correct tech stack with the currently supported versions of the code. So that you can focus on writing services without thinking about these things

Who ?

Are you an early stage startup who wants to focus on developing industrial strength services using well established patterns without bothering to find the optimal tech stack or browsing through reams of documentation? If yes, Chenile can help you there

Are you a Spring Boot developer who is confused in perusing the documentation of different frameworks ?

Patterns & Blue Prints

Interested in developing Moduliths? Chenile has its own pattern for developing robust moduliths.

Chenile has its own blue print to develop modulith services using the CQRS pattern.

Best of all Chenile ships with a code generator that generates code that complies to these blue prints. This code can give a huge jumpstart.

Our workflow , query and interceptor blue prints are easy to understand and ship with documentation. These are generated by the code generator

Extensible

Chenile is SOLID compliant. The framework is built to be extensible. It is trivial to write new blue prints and integrate with chenile. The code generator can be customized to generate code compliant to these blue prints.

Implementing horizontal concerns in the last mile is a killer feature with Chenile. Interceptors can be introduced before services are invoked. These can implement a lot of features.

Chenile also ships with configurable options for most of its features. These can be used to extend the framework and customize the services.

Features

HTTP

All services are exposed by HTTP transports. HTTP proxy is also included

i18n

Integrated and enforced out of the box.

Error Handling

Internationalized error messages. Support for Warnings.

Configuration Management

Cascading configurations with environment specific overrides

State Management

Externalized state management configuration.

Flexible Middlewares

Default middlewares to address specific horizontal concerns like transformation, input validation etc.

Version Management

Version control management for Micro services integrated

Sophisticated DevOps

Docker integration. Default Makefiles.

BDD

Cucumber integrated. BDD & full HTTP black box testing supported out of the box
